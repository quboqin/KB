## 6. 2.4G 设备网关分流配置

在主路由+旁路由的架构下，旁路由 (`192.168.123.99`) 接管了全局 DHCP，所有 2.4G 设备默认指向旁路由。为保证 IoT 设备稳定，需要让这些设备直接指向主路由 (`192.168.123.6`)。

### 方法一：旁路由 DHCP 静态绑定 + 自定义网关（强烈推荐）

通过 OpenWRT 的 dnsmasq 服务，在下发 IP 时将网关替换成主路由。

**配置步骤：**

1. 登录旁路由 (`192.168.123.99`) 后台
2. 导航到 **网络 > DHCP/DNS > 静态地址分配**
3. 添加设备的 MAC 地址（如小米空调 `7C:C2:94:31:1A:75`），固定 IP（如 `192.168.123.104`）
4. 在 DHCP-Options 字段输入：

```
3,192.168.123.6
6,223.5.5.5,114.114.114.114
```

- `3` = 网关（Router Option），强制指向 `.6`
- `6` = DNS，指定阿里云/114 DNS

5. 保存并应用，重启智能设备即可生效

### 方法二：旁路由 Passwall ACL 访问控制（最省事）

网关依然指向旁路由，但在旁路由内部放行 IoT 设备流量。

1. 登录旁路由，进入 **Passwall > 访问控制**
2. 将智能家居 IP 的代理模式设置为**直连/不代理**

> 进阶建议：将 IoT 设备 IP 分配在 `192.168.123.200-250` 段，在 Passwall 对整段直连。
>
> 缺点：旁路由断电时 IoT 设备也会断网。

### 方法三：设备端手动指定静态 IP（适用于手机/电脑）

1. 在设备 Wi-Fi 设置中切换为**静态 IP**
2. 网关/路由器填 `192.168.123.6`，DNS 填 `223.5.5.5`

> 缺点：大多数 IoT 设备没有手动 IP 设置选项，仅限手机/PC。

### 终极方案建议

- DHCP 范围 `100-200`：手机/电脑/Apple TV，走 `.99` 网关和透明代理
- 静态绑定 `201-250`：IoT 设备，DHCP Option 3 下发 `.6` 网关
- 实现"科学网络"与"IoT 网络"的逻辑隔离

## 7. 局域网访问桥接光猫后台

针对"光猫桥接 + 小米路由拨号"的网络环境，以下提供三种访问光猫 (`192.168.1.1`) 的方案。

### 方案一：旁路由 NAT 映射（最优雅，随时随地访问）

**前置条件**：一根网线从光猫 LAN2 连到主路由/交换机 LAN 口。

**OpenWRT 旁路由配置：**

1. 登录旁路由 (`192.168.123.99`)
2. 进入 **网络 > 接口 > 添加新接口**
   - 名称：`Modem`
   - 协议：静态地址
   - 接口/设备：`br-lan`
3. 基本设置：
   - IPv4 地址：`192.168.1.254`
   - 子网掩码：`255.255.255.0`
   - 不填网关
4. 防火墙设置：
   - 若有 `wan` 区域：选择 `wan`（默认带 NAT）
   - 若无 `wan` 区域：新建 `modem_zone`，勾选 **IP 动态伪装 (Masquerading)**，允许从 `lan` 转发到 `modem_zone`

配置完成后，网关指向旁路由的设备直接访问 `http://192.168.1.1` 即可打开光猫后台。

> 注意：确保光猫桥接模式下已关闭自身 DHCP 服务，避免分配错误 IP 导致断网。

### 方案二：设备手动改 IP（适合临时操作）

1. 电脑用网线直连光猫 LAN2
2. 网络适配器设置静态 IP：`192.168.1.2`，子网 `255.255.255.0`，网关 `192.168.1.1`
3. 浏览器访问 `http://192.168.1.1`
4. 完成后改回 DHCP 自动获取

### 方案三：手机连光猫 Wi-Fi（最简单）

1. 查看光猫背面贴纸，找到 Wi-Fi 名称（`ChinaNet-xxxx`）和密码
2. 手机连接该 Wi-Fi
3. 浏览器访问 `http://192.168.1.1`

> 配置完建议关闭光猫 Wi-Fi 避免信号干扰。
