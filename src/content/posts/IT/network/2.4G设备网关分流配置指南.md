📡 2.4G 设备网关分流与优化指南

在主路由+旁路由的架构下，由于旁路由 (192.168.123.99) 接管了全局 DHCP，所有接入 2.4G Wi-Fi 的设备默认都会指向旁路由。

为了保证智能家居（IoT）设备的绝对稳定，我们通常需要让这些设备直接指向主路由 (192.168.123.6)。以下是三种实现方法，按推荐程度从高到低排列：

方法一：旁路由 DHCP 静态绑定 + 自定义网关（🌟 强烈推荐）

这是最优雅、最彻底的解决方案。由于智能家居设备自身无法手动修改 IP 和网关，我们可以通过旁路由（通常是 OpenWRT）的 dnsmasq 服务，在下发 IP 时，悄悄把网关替换成主路由的 IP。

配置步骤（以 OpenWRT 为例）：

**登录旁路由 (192.168.123.99)**后台。

导航到 网络 (Network) -> DHCP/DNS -> 静态地址分配 (Static Leases)。

添加设备：

输入您设备的 MAC 地址（例如小米空调 7C:C2:94:31:1A:75）。

固定一个 IP 地址（例如 192.168.123.104）。

关键步骤（高级设置/DHCP 选项）：

找到该条目的 DHCP-Options（DHCP 选项）字段。

输入以下代码（注意逗号和数字）：

3,192.168.123.6
6,223.5.5.5,114.114.114.114


原理解释： - 3 代表网关 (Router Option)，将这个设备的网关强制指定为 .6。

6 代表 DNS (DNS Option)，将这个设备的 DNS 指定为阿里云/114，防止 DNS 污染。

保存并应用。重启您的智能设备（断电重连），它就会自动获取到 .6 的网关，彻底脱离旁路由的控制！

方法二：旁路由 Passwall ACL 访问控制（⚡ 最省事）

如果您觉得给几十个智能家居设备挨个绑定 MAC 地址太麻烦，您可以选择**“网关依然指向旁路由，但在旁路由内部放行”**的懒人做法。

流量依然会经过旁路由的网口，但不会经过代理核心（不走 Passwall），从而降低延迟。

配置步骤：

登录旁路由，进入 Passwall 设置页面。

导航到 访问控制 (Access Control) 选项卡。

找到 局域网内设备代理模式 (LAN Proxy Mode)。

将智能家居的 IP 地址（如 192.168.123.104, .145 等）添加到列表中。

将这些 IP 的代理模式强制设置为 直连 (Direct) 或 不代理 (No Proxy)。

进阶建议： 您可以利用小米主路由的 DHCP 静态绑定功能（如果能用），把智能家居的 IP 全部分配在 192.168.123.200 - 250 之间，然后在 Passwall 里直接对这个 IP 段设置直连，一劳永逸。

⚠️ 缺点：如果旁路由断电关机，智能家居依然会断网。

方法三：设备端手动指定静态 IP（📱 适用于手机/电脑）

如果是您的个人设备（比如一台连着 2.4G 频段的旧笔记本、或者某台测试手机），不需要走代理，可以直接在设备端修改。

配置步骤（以手机为例）：

连接上您的 2.4G Wi-Fi。

点击 Wi-Fi 名称右侧的 i (信息) 或 设置 齿轮。

找到 IP 设置，将 DHCP (自动) 切换为 静态 (Static)。

手动填写信息：

IP 地址：192.168.123.xxx（避开已用 IP）

子网掩码/前缀：255.255.255.0 或 24

路由器/网关：👉 192.168.123.6

DNS：223.5.5.5

保存即可。这台设备从此通过主路由直接上网，与旁路由无关。

⚠️ 缺点：绝大多数无屏幕的智能 IoT 设备（如灯泡、插座、窗帘）的 APP 里根本没有提供手动设置 IP 的选项，因此本方法仅限手机/PC。

💡 架构师总结建议

对于您的 HomeLab 和智能家居混合网络，我最推荐的终极方案是 方案一 + IP 段规划：

在旁路由的 DHCP 设置中，将分配范围限制在 100 - 200（供手机、电脑、Apple TV 动态获取，默认走 .99 网关和透明代理）。

把所有智能家居的 MAC 地址在旁路由里设置为静态绑定（比如分配到 201 - 250），并利用 DHCP Option 3 强行下发 192.168.123.6 的网关。

这样既做到了 “科学网络”与“IoT网络”的逻辑隔离，又保证了只要主路由不断电，就算你在 PVE 里把旁路由虚拟机删了，家里的灯和空调照样能用语音控制！
